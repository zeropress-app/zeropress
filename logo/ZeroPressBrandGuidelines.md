# ZeroPress Brand Guidelines (Draft v0.1)

## 1. Philosophy

ZeroPress is built around a simple principle:

**Publish durable artifacts, not runtime services.**

ZeroPress follows a **static-first, file-native architecture** where the final output of publishing is a set of durable HTML documents.

Key ideas behind the project:

* **Static-First** — Pages are generated at publish time.
* **File-Native** — Content is stored and delivered as files.
* **Durable HTML** — Published pages remain usable even when backend systems fail.

ZeroPress is designed for **longevity, clarity, and operational simplicity**, prioritizing content survivability over runtime complexity.

---

## 2. Logo Meaning

The ZeroPress logo uses a **cube symbol** to represent the core idea of the system: **published artifacts**.

The cube symbolizes:

* **Artifacts** — Final published outputs
* **Structured content** — Content assembled into deterministic results
* **Reproducible publishing** — The same inputs produce the same outputs
* **Durable documents** — Files that outlive frameworks, APIs, and runtimes

ZeroPress does not serve dynamic APIs.

It produces **files that can exist independently of the system that created them**.

The cube visually represents this philosophy.

---

## 3. Color System

Primary color usage follows a **teal → green gradient**, representing clarity, structure, and technical precision.

Recommended primary gradient range:

* Teal
* Cyan
* Soft green

This palette aligns with modern infrastructure tooling while remaining visually neutral and adaptable.

### Monochrome Versions

The logo must also work in **single-color environments**.

Supported monochrome variants:

* White (#FFFFFF)
* Black (#000000)

All official logo variants must remain recognizable in monochrome usage.

---

## 4. Logo Usage Principles

To maintain visual clarity and consistency:

* Maintain clear space around the logo equal to **at least half the logo height**.
* Avoid placing the logo on backgrounds that reduce contrast or legibility.
* The minimum recommended size for the symbol-only mark is **24px**.
* Use the symbol-only version for small contexts such as:

  * favicons
  * CLI tools
  * repository icons
  * compact UI elements

For primary branding contexts, use the **symbol + wordmark** combination.
